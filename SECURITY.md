# Security Policy

## Supported Branches

Security updates are applied to the default branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead, open a private vulnerability report through GitHub Security Advisories
for this repository (or contact the maintainer directly if advisories are not enabled).

Include:

- Affected component or path
- Reproduction steps
- Expected vs. observed behavior
- Severity assessment
- Suggested mitigation (if known)

You can also review implementation details in [docs/SECURITY.md](./docs/SECURITY.md).
